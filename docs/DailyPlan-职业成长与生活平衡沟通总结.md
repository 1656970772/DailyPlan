# DailyPlan：职业成长与生活平衡沟通总结

## 文档说明

- 日期：2026-03-13
- 目的：沉淀本次关于职业发展、技术学习、独立游戏、伴侣关系、健身与时间分配的沟通结论，作为后续行动参考。

## 一、AI 时代是否还要认真学虚幻引擎

结论：要学，而且面试依然看，但学习方式要升级。

- 虚幻引擎岗位仍然普遍看重 `C++`、`UE4/UE5`、玩法系统、性能优化、多人同步、调试与协作能力。
- AI 可以大幅提高资料检索、样板代码生成、报错分析和文档整理效率，但不能替代工程判断。
- 面试重点已经不只是“会不会用 UE”，而是“能不能把复杂功能做成、做稳、做快、做清楚”。

换句话说，需要从“背 API、记节点”转向“项目落地、架构判断、性能与联机、问题排查”。

## 二、5 年 Unreal 程序员的核心竞争力

对于 5 年经验的社招候选人，面试官通常重点看三件事：

1. 能不能独立扛模块
2. 有没有工程深度
3. 能不能提升团队效率

对应的能力重点如下：

- `C++` 基础是否扎实，是否能写出可维护、可扩展代码
- 是否理解 `Gameplay Framework`
- 是否能把蓝图与 `C++` 做出合理边界
- 是否能处理性能优化、崩溃排查、多人同步、异步加载等高频问题
- 是否能把需求拆解、识别风险、推动协作、带动初中级同学

## 三、社招面试清单

### 必须会

- `C++`
- `UE Gameplay Framework`
- 蓝图与 `C++` 分工
- 对象生命周期、反射、GC
- 调试定位
- 性能分析
- 基础架构设计

### 高频深水区

- 网络复制与状态同步
- 异步加载与资源管理
- 动画状态、事件驱动与联动逻辑
- 组件化设计
- 卡顿和崩溃排查
- 内存与对象管理

### 5 年经验的真正考点

- 是否能独立推进需求
- 是否能做技术取舍
- 是否能处理线上和版本风险
- 是否能提高团队整体效率

## 四、没有完整模块机会时，如何弥补短板

如果暂时没有机会担任完整模块负责人，不代表无法证明能力。可以通过以下方式补足证据链：

### 1. 把局部功能讲成闭环

即使只负责了一个子系统中的一部分，也要能说清楚：

- 背景和目标是什么
- 问题在哪里
- 自己负责哪部分
- 试过哪些方向
- 为什么最终选择当前方案
- 最终结果如何

### 2. 主动做小而完整的技术 Demo

推荐方向：

- `GAS / 战斗技能`
- `AI 行为树 + 感知`
- `联机同步小玩法`
- `背包 / 任务 / 存档`
- `编辑器工具`

要点不是体量大，而是闭环完整、设计清晰、可讲解。

### 3. 做技术专题而不是空等机会

可以围绕以下主题独立补强：

- 性能优化案例
- 网络同步案例
- 动画系统案例
- 加载与内存案例
- 编辑器工具或流程提效案例

### 4. 准备四类材料

- `2 个` 小而完整的个人 Demo
- `3 个` 真实项目案例复盘
- `1 份` 性能 / 联机 / 架构专题总结
- `1 份` 源码阅读或工具项目说明

## 五、社招 Demo 还有没有用

结论：有用，但对 5 年经验程序员来说，`Demo 是补充，不是主履历`。

- Demo 不能替代真实项目经验
- Demo 很适合弥补“缺少完整 owner 经历”“项目保密难讲透”“只做过零散功能”等问题
- 对中高级候选人，最有价值的 Demo 通常不是玩法小玩具，而是技术型 Demo

最值得做的 Demo 类型：

- 系统 Demo：战斗框架、任务系统、存档、组件化交互
- 联机 Demo：复制、预测、延迟处理、状态同步
- 性能 Demo：对象池、异步加载、Profile 前后对比
- 工具 Demo：编辑器插件、内容生产提效工具
- 源码向 Demo：基于 UE 机制做扩展

原则：`一个小而硬的 Demo，胜过多个花哨但浅的 Demo。`

## 六、项目细节忘了怎么办

这是正常现象。复杂问题的解决过程天然会随时间模糊，关键不是强行记全，而是重新提取“可复述结构”。

### 回忆顺序

- 问题是什么
- 影响了谁
- 为什么难
- 自己负责什么
- 试过哪些方向
- 为什么选择当前方案
- 结果如何

### 反向恢复记忆的材料

- `Git commit`
- `PR / Code Review`
- `Jira / 禅道 / TAPD`
- 聊天记录
- 报错截图
- 性能分析截图
- 设计文档
- 日志关键词

### 项目复盘模板

每个案例可以压缩成一页：

- 背景
- 问题
- 难点
- 我的职责
- 方案对比
- 最终方案
- 结果
- 反思

面试时允许忘掉部分实现细节，但不能忘掉核心判断逻辑。

## 七、独立游戏、技术推进、感情生活、健身同时存在时怎么排

当前面临的是多条重要主线并存：

- 职业上升
- 独立游戏开发
- 伴侣关系与结婚准备
- 身体建设

核心结论：

`不能四条线同时高速推进，但可以一条主线推进，另外几条不断线。`

建议使用三层目标法：

### 主目标

一个阶段只保留 `1 个` 主目标，例如：

- 找更好的工作
- 做独立游戏原型
- 推进关系与结婚准备

### 副目标

保留 `1-2 个`，低速推进。

### 保底目标

只求不断线，不求突破，例如：

- 每周健身 `2-3` 次
- 每周做一次技术学习
- 每周推进一点个人项目

建议策略：

- 现实生活优先级不能长期后置
- 主业能力要稳步推进
- 独立游戏先做超小原型
- 健身先走维持策略

## 八、AI 工具太多，学不过来怎么办

结论：不需要追所有工具，只需要建立一套真正有产出的最小工作流。

不应该“按工具学”，而应该“按任务学”。对于 Unreal 程序员，AI 主要服务这些任务：

- 查资料与读源码
- 写样板代码与重构辅助
- 排查报错与性能问题
- 整理技术文档、复盘、简历材料
- 辅助做 Demo 与原型

优先保留三类能力：

- 通用对话 / 推理
- 代码助手
- 自动化 / 检索

建议原则：

- 一段时间深用 `1-2` 个工具即可
- 每周只给 AI 学习 `2 小时`
- 必须和现实任务绑定，不能只做收藏和浅尝

目标不是“知道很多工具”，而是“形成自己的 AI 增强工作流”。

## 九、在中国、工作忙、社交少的情况下如何找伴侣

结论：不适合走“高频广撒网”的社交策略，更适合 `低频、高质量、可持续` 的方案。

### 主渠道：熟人介绍

让可信的人知道自己现在是认真找对象、以结婚为目标。
这通常比盲目陌生社交更高效。

### 第二渠道：官方或半官方联谊

重点关注：

- 工会
- 团委
- 妇联
- 人才办
- 园区
- 校友会
- 行业协会

### 长期渠道：固定兴趣场景

优先选择能重复见面的场景，例如：

- 羽毛球
- 徒步
- 跑团
- 桌游
- 读书会
- 摄影
- 志愿者活动

### 线上平台：作为辅助

- 每天控制在 `15-20` 分钟
- 聊 `3-7` 天感觉合适就约线下见面
- 不长聊、不养鱼、不同时铺太多线

### 忙人可执行的周节奏

- 周中两次轻量推进消息和介绍
- 周末一次线下活动
- 周末预留一次单独见面时间

目标不是短期认识很多人，而是稳定认识合适的人。

## 十、当前最适合的总体策略

结合本次沟通，当前更适合采取以下思路：

1. 以 `主业能力提升` 作为长期底盘
2. 用 `小而硬的技术 Demo` 补足完整模块经历不足的问题
3. 用 `项目复盘材料` 对抗记忆流失，提前准备面试表达
4. 把独立游戏控制在可验证的小原型级别
5. 把 AI 当加速器，而不是新的主线任务
6. 用低频高质量的方式推进伴侣关系，不再等待“以后有时间再说”
7. 接受当前阶段必须做取舍，而不是追求所有方向同时冲刺

## 十二、核心提醒

- Unreal 仍然值得深学，但要从“会用”升级到“能负责、能判断、能落地”
- 社招看重的是证据链，不只是工作年限
- 忘项目细节并不可怕，关键是能把经历提炼成清晰判断
- 当前不是时间管理失败，而是进入了必须做阶段性取舍的人生阶段
- 不需要追所有 AI 工具，只需要用好少数真正提升产出的工具
- 找伴侣不一定靠高频社交，低频高质量更适合当前状态
